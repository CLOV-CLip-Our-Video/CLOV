## 오늘의 배운점/생각
- KEYS 명령어의 문제점
    - O(N) 시간복잡도: 모든 키를 순차 스캔
    - 블로킹 연산: 실행 중 다른 Redis 작업 대기
    - 키 개수에 비례하여 성능 저하
- SCAN 명령어의 장점
    - Non-blocking: 다른 Redis 작업과 인터리빙 가능
    - 점진적 스캔: 작은 단위로 나누어 처리
    - 메모리 효율성: 대용량 키셋도 안정적 처리
