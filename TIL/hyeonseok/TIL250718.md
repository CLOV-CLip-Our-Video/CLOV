## 느낀 점 및 배운 점

### 1. 인라인 스타일의 Tailwind 와 module.css 파일의 조합
#### 인라인 유틸리티 클래스(권장)
Tailwind는 일반적으로 인라인 유틸리티 클래스 사용을 권장한다.
```jsx
function Button() {
  return (
    <button className="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded">
      클릭하세요
    </button>
  );
}
```
이러한 인라인 스타일 Tailwind를 사용하면 다음과 같은 이점이 있다.
- **빠른 개발**: CSS 파일을 왔다갔따 할 필요 없음
- **유지보수**: 컴포넌트와 스타일이 한 곳에 있어서 관리하기 쉬움
- **사용하지 않는 CSS 제가**: 실제로 사용한 클래스만 빌드에 포함됨
- **일관성**: 디자인 시스템이 내장되어 있어 일관된 스타일 가능

#### 긴 클래스명 정리 팁
클래스명이 길어지면 이렇게 정리할 수도 있다.
```jsx
// 조건부 스타일링
const buttonClass = `
  bg-blue-500 hover:bg-blue-700 
  text-white font-bold 
  py-2 px-4 rounded
  transition duration-300
`;

// 또는 clsx 라이브러리 사용
import clsx from 'clsx';

const buttonClass = clsx(
  'bg-blue-500 hover:bg-blue-700',
  'text-white font-bold',
  'py-2 px-4 rounded',
  isActive && 'ring-2 ring-blue-300'
);
```
#### CSS 모듈로 하는 경우
다만 Tailwind는 만능이 아니며, 코드의 간결성을 해칠 수도 있다. 이런 문제를 해결하기 위해 CSS 모듈을 사용해볼 수 있다.
1. **복잡한 애니메이션이나 특수 효과**
```jsx
/* Button.module.css */
.slideIn {
  animation: slideIn 0.3s ease-out;
}

@keyframes slideIn {
  from { transform: translateX(-100%); }
  to { transform: translateX(0); }
}

.customShadow {
  box-shadow: 0 10px 25px rgba(0, 0, 0, 0.1), 0 4px 10px rgba(0, 0, 0, 0.08);
}
```
2. **컴포넌트 특화 스타일**
```jsx
/* Card.module.css */
.cardContainer {
  /* 이 카드 컴포넌트만의 특별한 스타일 */
  background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
  backdrop-filter: blur(10px);
}

.cardTitle {
  /* 특정 폰트나 복잡한 텍스트 스타일 */
  font-family: 'Noto Sans KR', sans-serif;
  background: linear-gradient(45deg, #fff, #ddd);
  -webkit-background-clip: text;
  -webkit-text-fill-color: transparent;
}
```

### 2. 페이지가 먼저인가 컴포넌트가 먼저인가
혼자 작업할 때는 페이지를 먼저 구현 시작하면서 필요할 때마다 컴포넌트를 만드는 방식이 편했다. 하지만 협업할 때는 conflict도 생각해야하며, 효율성을 중요시해야하기 때문에 각종 기본 컴포넌트들을 먼저 완성하는 것이 더 낫다고 판단했다.
